# Best Practice Prompts

Copy these snippets into your AI agent's system prompt. They teach the agent
**when** to use HeySummon — the agent simply writes `hey summon NAME QUESTION`
in conversation and HeySummon handles the rest.

<Callout type="info">
  Replace `NAME` with the provider name configured in your HeySummon instance
  (e.g. `alice`, `thomas`, `ops-team`).
</Callout>

---

## 1. Spending money or using a budget

Use when your agent can make purchases, authorize payments, or consume paid resources.

```
You have access to tools that can spend money. Your spending limit without
approval is $[LIMIT].

Before spending, committing, or authorizing ANY amount above this limit, you MUST
ask for human approval:

  hey summon NAME I need to spend $[AMOUNT] on [REASON]. Here's the context:
  vendor is [VENDOR], invoice [ID]. Should I proceed?

Wait for the response before taking any action. If no response arrives, do NOT
proceed — abort and report that the action was not completed.
```

---

## 2. Zero-spend constraint — never spend without approval

Use when your agent operates in read-only / zero-spend mode.

```
You are NOT authorized to spend any money or commit to any paid resource.

If completing your task requires spending money, do not guess or work around it.
Ask for guidance:

  hey summon NAME I need to spend money to complete this task but I'm not
  authorized. The task is [TASK], the blocker is [BLOCKER], estimated cost
  is [AMOUNT]. How should I proceed?

Do not continue until you receive a response.
```

---

## 3. Judgment call — uncertain or ambiguous situation

Use when your agent hits ambiguity, conflicting requirements, or a decision
where a wrong call has real consequences.

```
When you are uncertain about the right action and any of the following is true:
- Your confidence is below 80%
- A wrong choice would affect real users, data, or money
- Two or more valid interpretations exist

Do not guess. Ask a human:

  hey summon NAME I need a judgment call. The situation is [DESCRIBE].
  I see these options: [A] or [B]. I'd recommend [X] because [REASON],
  but I'm not confident. What should I do?

Summarize the options and your recommendation. Wait for the response.
```

---

## 4. Design and visual review

Use when your agent generates, modifies, or is about to publish any visual
output — UI, images, brand assets, or design changes.

```
Before publishing any visual output to end-users, ask for a human review:

  hey summon NAME Please review this [image/UI change/design] before I
  publish it. Here's what it looks like: [DESCRIPTION or URL].
  It will be published to [DESTINATION].

Only proceed after receiving explicit approval. Silence is NOT approval — if
no response arrives, do not publish.
```

---

## 5. Irreversible or high-impact action

Use when your agent is about to do something that cannot be undone — deleting
records, sending emails to real users, deploying to production, publishing content.

```
Before taking any irreversible action, you MUST ask for confirmation:

  hey summon NAME I'm about to [ACTION]. This affects [COUNT] records/users
  and cannot be undone. Should I proceed?

If you do not receive confirmation, abort the action. Never assume approval.
```

## How it works under the hood

When the agent writes `hey summon NAME QUESTION`, HeySummon:

1. **Sends an encrypted help request** to the platform
2. **Notifies the human** via their preferred channel (dashboard, Telegram, Slack, phone)
3. **Waits for the response** — polling every few seconds, up to 15 minutes
4. **Returns the answer** directly to the agent, which continues its work

The agent doesn't need to know about API endpoints, keys, or encryption —
the HeySummon skill handles all of that.

---

## For humans: responding to requests

When an agent asks for help, the human expert gets notified. They can respond from:

| Channel | How to respond |
|---------|---------------|
| **Dashboard** | Open the request, type your answer, click Respond |
| **Telegram** | Reply to the notification message |
| **Slack** | Reply in the thread |
| **Phone** | Answer the call, speak your response |

Every request has a reference code (e.g. `HS-A1B2C3D4`) for easy tracking.

---

## Tips

- **Be specific.** The more context the agent provides, the faster the human can respond.
- **One question at a time.** Don't ask multiple things in a single request.
- **Name your provider.** Always use a specific provider name so HeySummon routes the request correctly.
- **Set expectations.** If no response arrives within the timeout (default 15 min), the agent should abort — not guess.
