# OpenClaw

HeySummon integrates with OpenClaw through the ClaWhub marketplace. OpenClaw agents can install the skill directly from ClaWhub and register experts using API keys.

## Installation

### Via Setup Link (Recommended)

Your expert provides a setup link from the HeySummon dashboard. Run the install command from the directory that holds (or will hold) your `skills/heysummon/` folder -- typically the OpenClaw skill root. The install runs from your current working directory and no longer assumes `~/clawd`:

```bash
curl -fsSL "https://your-instance.com/api/v1/setup/st_your_token_here/install.sh" | bash
```

The install script is atomic: it stages skill scripts in a sibling temp directory, verifies sha256 checksums, swaps them into `skills/heysummon/`, writes `.env` (single-quoted values so multi-line summon contexts round-trip cleanly), registers your expert, and proves a real round-trip before declaring success. On any failure it rolls back. See [Install Script Output](/clients/install-output) for the structured JSON schema, exit codes, and a `jq` recipe for piping the result into agent memory.

#### Idempotency & re-running

The install is idempotent:

- If `skills/heysummon/.env` exists with the **same** `HEYSUMMON_API_KEY`, the install exits 0 and prints the current success JSON without overwriting anything.
- If `.env` exists with a **different** `HEYSUMMON_API_KEY`, the install refuses by default (exit `15`). Re-run with `HEYSUMMON_FORCE=1` to replace:

  ```bash
  HEYSUMMON_FORCE=1 curl -fsSL "https://your-instance.com/api/v1/setup/st_token/install.sh" | bash
  ```

This protects against the previous failure mode where a half-completed install (different key, partial scripts) silently overwrote secrets.

### Via ClaWhub (alternative)

If you prefer the ClaWhub installer for the skill files, you can still run:

```bash
npx clawhub@latest install heysummon
```

…and then run the setup command above to register your expert and write `.env`.

### OpenClaw Sub-Channels

OpenClaw supports sub-channels for different messaging platforms:
- **Telegram** -- expert receives notifications via Telegram bot
- **WhatsApp** -- expert receives notifications via WhatsApp

The sub-channel is configured when the expert creates the setup link.

## Usage

Once installed, the OpenClaw agent detects the skill and can ask for help:

```
hey summon Thomas should I use PostgreSQL or SQLite for this project?
```

Or request approval:

```
hey summon Thomas I want to refactor the auth module. Please approve.
```

### Notify mode (one-way, non-blocking)

For fire-and-forget alerts -- deploy finished, long job done, status heads-up -- use `notify`:

```
notify Thomas deploy finished: build 42 is live on prod
```

The expert sees a single **Acknowledge** button (Telegram, WhatsApp, or the
dashboard) and the agent continues immediately. Notifications expire after 7
days if nobody acks. See
[Consumer API Reference -> Notification Mode](/consumer/api-reference#notification-mode-notify).

## Expert Skill

OpenClaw also supports an **expert skill** (`heysummon-expert`) that allows an OpenClaw agent to act as the expert -- receiving and responding to help requests. Install it from ClaWhub:

```bash
npx clawhub@latest install heysummon-expert
```

This sets up a polling watcher that monitors for incoming requests and presents them to the agent for response.

## Skill Location

```
~/clawd/skills/heysummon/
  SKILL.md           # Instructions for the agent
  scripts/           # ask.sh, setup.sh, etc.
  .env               # API key and configuration
  experts.json       # Registered experts
```

## Configuration

The `.env` file in the skill directory contains:

```env
HEYSUMMON_BASE_URL=https://your-instance.com
HEYSUMMON_API_KEY=hs_cli_your_key_here
HEYSUMMON_TIMEOUT=900
HEYSUMMON_POLL_INTERVAL=3
HEYSUMMON_SUMMON_CONTEXT=Only summon when stuck or need approval
```

| Variable | Description | Default |
|----------|-------------|---------|
| `HEYSUMMON_BASE_URL` | Your HeySummon instance URL | `http://localhost:3425` |
| `HEYSUMMON_API_KEY` | Client API key (starts with `hs_cli_`) | Required |
| `HEYSUMMON_TIMEOUT` | Request timeout in seconds | `900` (15 min) |
| `HEYSUMMON_POLL_INTERVAL` | How often to check for a response (seconds) | `3` |
| `HEYSUMMON_SUMMON_CONTEXT` | When to summon the expert (set by expert) | Optional |

## Commands

```bash
# Ask a question
bash scripts/ask.sh "Your question" "" "ExpertName"

# Add an expert
bash scripts/add-expert.sh "hs_cli_key" "ExpertName"

# List experts
bash scripts/list-experts.sh

# Check request status
bash scripts/check-status.sh HS-ABCD
```
