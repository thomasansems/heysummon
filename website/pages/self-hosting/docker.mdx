# Docker

The recommended way to self-host HeySummon. Runs the app and PostgreSQL in Docker.

## Prerequisites

- Docker 24+
- Docker Compose v2

## Quick start

One command — no source code, no manual configuration:

```bash
curl -fsSL https://raw.githubusercontent.com/thomasansems/heysummon/main/install.sh | bash
```

This script:
1. Creates `~/.heysummon-docker/`
2. Downloads `docker-compose.yml`
3. Generates `NEXTAUTH_SECRET` and `DB_PASSWORD` automatically
4. Writes a ready-to-use `.env`
5. Runs `docker compose up -d`

Open `http://localhost:3445` when done.

## Docker image

```
ghcr.io/thomasansems/heysummon
```

Pull manually:

```bash
docker pull ghcr.io/thomasansems/heysummon:latest
```

## Architecture

```
Internet
    |
    v
App (:3445)            — API, dashboard, auth, content safety
    |
    +---> PostgreSQL   — data storage
```

Content safety (HTML sanitization, URL defanging, PII detection/blocking) runs as built-in middleware inside the app.

## Making it public

The installer asks how you want to expose HeySummon. You can switch later by editing `.env` and re-running `docker compose --profile <name> up -d`.

| Option | Best for | What you need |
|--------|----------|---------------|
| **Direct port** | Quick LAN / VPN testing | Just an open port |
| **Caddy + HTTPS** *(recommended for production)* | Public deployment on a VPS / EC2 | A domain you control |
| **Cloudflare Tunnel** | You already use Cloudflare DNS | A Cloudflare account |
| **Tailscale Funnel** | Free public URL, no domain | A Tailscale account |

### Caddy + automatic HTTPS

[Caddy](https://caddyserver.com) is a small reverse proxy that runs alongside HeySummon and **automatically obtains and renews a real Let's Encrypt certificate** for your domain. No certbot, no cron jobs, no renewal management.

It's the simplest way to put HeySummon on the public internet with a real HTTPS certificate.

#### Requirements

1. A domain name you control (e.g. `heysummon.example.com`)
2. A DNS **A-record** pointing that domain to your server's public IP
3. Inbound TCP ports **80** and **443** open on your firewall / security group

<Callout type="warning">
  Let's Encrypt does **not** issue certificates for raw IP addresses. You need a real domain name (or subdomain). If you don't have one, see Cloudflare/Tailscale below — they give you HTTPS without a domain of your own.
</Callout>

#### DNS setup

Log in to your domain registrar (Namecheap, Cloudflare, GoDaddy, TransIP, etc.) and add a single A-record:

| Type | Name / Host | Value | TTL |
|------|-------------|-------|-----|
| `A` | `heysummon` *(or whichever subdomain you want)* | `<your server's public IP>` | `300` |

**Notes:**
- Most registrars want only the subdomain in the "Name" field. Some want the full hostname (`heysummon.example.com`). Check your provider's docs.
- For the apex domain (no subdomain), use `@` as the name.
- On AWS EC2, allocate an **Elastic IP** to your instance before setting up DNS — otherwise the public IP changes every reboot.

Verify the record propagated:

```bash
dig +short heysummon.example.com
# Should print your server's IP
```

#### Start Caddy

```bash
cd ~/.heysummon-docker

# Add your domain to .env
echo 'DOMAIN=heysummon.example.com' >> .env

# Update the public URLs to use HTTPS
sed -i 's|NEXTAUTH_URL=.*|NEXTAUTH_URL=https://heysummon.example.com|' .env
sed -i 's|HEYSUMMON_PUBLIC_URL=.*|HEYSUMMON_PUBLIC_URL=https://heysummon.example.com|' .env

# Start with the caddy profile
docker compose --profile caddy up -d

# Watch Caddy obtain the certificate (10-30 seconds)
docker compose --profile caddy logs -f caddy
```

You should see `certificate obtained successfully`. Then `https://heysummon.example.com` works with a real Let's Encrypt cert, auto-redirects from HTTP, and renews itself every 60 days.

#### What if it fails?

| Error in Caddy logs | Cause | Fix |
|---------------------|-------|-----|
| `no such host` | DNS not propagated yet | Wait 1-5 minutes, run `dig` to verify |
| `connection refused` on `:80` | Firewall blocks port 80 | Open inbound TCP 80 + 443 in your security group |
| `challenge failed` | DNS points elsewhere | Verify A-record matches your server's actual public IP |
| `unauthorized` | Cloudflare proxy on (orange cloud) | Set the Cloudflare DNS record to "DNS only" (grey cloud), or use Cloudflare Tunnel instead |

### Cloudflare Tunnel

Use this if you already use Cloudflare DNS, or if you can't open ports 80/443. Cloudflare Tunnel uses an outbound connection from your server, so no inbound firewall rules are needed.

```bash
CLOUDFLARE_TUNNEL_TOKEN=your-token docker compose --profile cloudflare up -d
```

### Tailscale Funnel

Use this if you don't have a domain. Tailscale gives you a free `*.ts.net` subdomain with HTTPS included.

```bash
TAILSCALE_AUTHKEY=your-key docker compose --profile tailscale up -d
```

## Updating

```bash
cd ~/.heysummon-docker
docker compose pull
docker compose up -d
```

To pin a version:

```bash
HEYSUMMON_VERSION=0.1.0 docker compose up -d
```

## Logs

```bash
docker compose logs -f app
```

## Backups

```bash
# Backup
docker exec heysummon-db-1 pg_dump -U heysummon heysummon > backup.sql

# Restore
cat backup.sql | docker exec -i heysummon-db-1 psql -U heysummon heysummon
```

## Debug tools

```bash
# Prisma Studio — database browser at http://localhost:3447
docker compose -f docker-compose.dev.yml --profile debug up -d
```

<Callout type="info">
  Prisma Studio is only available in development builds (`docker-compose.dev.yml`). It never runs in the production compose file.
</Callout>
