# Dashboard

The HeySummon dashboard is the main interface for experts to manage requests, users, and keys.

---

## Pages

| Page | URL | Description |
|------|-----|-------------|
| **Requests** | `/dashboard` | All incoming help requests |
| **Users** | `/dashboard/experts` | Manage expert profiles + IP Security |
| **Clients** | `/dashboard/clients` | Manage client keys + IP Security |
| **API Keys** | `/dashboard/keys` | All API keys |
| **Audit Logs** | `/dashboard/audit-logs` | Full action history |
| **Settings** | `/dashboard/settings` | Account and instance settings |

---

## Requests list

The main page shows all requests sorted by status and time.

A filter pill row at the top of the list switches between:

- **All** — default; help requests and notifications mixed by recency
- **Help requests** — only requests expecting a reply (`responseRequired: true`)
- **Notifications** — only one-way alerts (`responseRequired: false`)

Under the **Notifications** filter, a secondary sub-tab toggles between
**Pending** (unacknowledged) and **Acknowledged** (last 30 days, for auditing).

### Card states

- **Pending** — waiting for a response (highlighted)
- **Reviewing** — someone has opened this request
- **Responded** — answered
- **Expired / Closed** — no longer active
- **Notification** — a muted card with a `Notification` badge, no unread count, and a single **Acknowledge** button. No reply surface.
- **Acknowledged** — terminal state for notification-mode requests; the card shows the `acknowledgedAt` timestamp.

Click any request to view the decrypted content and respond.

---

## Handling notifications

Notification-mode requests (`responseRequired: false`, submitted by consumers
via `notify()`) render differently from help requests:

- Single **Acknowledge** button, no reply form, no message history panel
- 7-day TTL instead of 72 hours
- Once you click **Acknowledge**, the card transitions to `acknowledged` and
  the consumer receives a `notification_acknowledged` event on their next poll
- Unacknowledged notifications auto-`expired` after 7 days

The same Acknowledge affordance is also available from Telegram (inline
button), Slack (Block Kit action), and OpenClaw (action webhook) -- see
[Responding to Requests](/expert/conversations#notifications).

## Users page (Experts)

The **Users** page lets you manage expert profiles. Each user can have:
- An expert key (`hs_exp_*`)
- IP bindings — visible inline with Allow/Blacklist/Remove actions
- An active/inactive status
- **Summoning context** is configured per-client during client setup (not on the expert profile). Recently used contexts are remembered for quick reuse.

---

## Clients page

The **Clients** page lets you manage client API keys. Features are identical to the Users page — IP Security is shown inline per client.

---

## Stats

The dashboard homepage shows real-time stats:
- Total requests
- Pending / Responded / Expired counts
- Average response time
- Active connections

  Stats are calculated live from the database — no caching. Refresh to get the latest counts.
</Callout>
