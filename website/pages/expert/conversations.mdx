# Responding to Requests

As an expert, you can respond to help requests directly from your messaging app — no need to open the dashboard.

---

## How it works

When an AI agent submits a request, you receive a notification:

```
🦞 HeySummon [HS-A1B2C3D4] New request
📝 Should I delete the production database?
```

The `[HS-XXXXXXXX]` tag is the **reference code** for that conversation.

---

## Responding via Telegram

### Option 1: Reply to the notification (recommended)

Reply to the notification message — your assistant (e.g. OpenClaw) detects the ref code from the quoted message and forwards your response automatically.

```
You reply: "No, never delete production databases."
       ↓
✅ Response sent to agent
```

### Option 2: Include the ref code in a new message

```
[HS-A1B2C3D4] No, never delete production databases.
```

Everything after the tag is sent as the response.

---

## Responding via dashboard

1. Open the dashboard
2. Click the request with the matching ref code
3. Read the decrypted question
4. Type and submit your response

---

## Approval buttons

When an AI agent submits a request with `requiresApproval: true`, you receive a notification with **Approve** and **Deny** buttons instead of a free-text prompt. This is used for budget decisions, destructive actions, or any binary yes/no question.

### Telegram

The notification appears with inline buttons directly in your chat:

```
*Approval required* HS-A1B2C3D4

*Question:* Should I proceed with the $5,000 purchase?

[ Approve ]  [ Deny ]
```

Tap a button to submit your decision. The message updates to confirm:

```
Request HS-A1B2C3D4 -- Decision: Approved
```

### Slack

The notification appears as a Block Kit message with styled buttons:

```
*Approval required* HS-A1B2C3D4

*Question:* Should I proceed with the $5,000 purchase?

[Approve]  [Deny]
```

The green "Approve" and red "Deny" buttons are interactive Slack actions. After you click, the message updates to show the decision and the buttons are removed.

### OpenClaw

OpenClaw receives a webhook notification with action URLs. Your OpenClaw assistant presents the approve/deny options through its own interface. The decision is sent back to HeySummon via a callback webhook.

### How it works

1. Consumer submits a request with `requiresApproval: true`
2. HeySummon sends approval buttons to your active notification channels
3. You tap Approve or Deny
4. The decision is recorded and the consumer receives it on the next poll
5. Each request can only receive one decision -- subsequent button presses are ignored

<Callout type="info">
  Approval requests that expire (72h) or are cancelled can no longer be approved or denied. The buttons will show the expired/cancelled status if pressed.
</Callout>

## Multi-turn conversations

Requests support multiple back-and-forth messages. Follow-ups share the same ref code:

```
🦞 HeySummon [HS-A1B2C3D4] Follow-up
📝 What should I do instead?
```

Respond the same way — the ref code links it to the same conversation thread.

---

## Full example

```
Agent ──── "Should I delete the DB?" ────▶ HeySummon
                                                │
                                      Notification to you:
                                      [HS-A1B2C3D4] Should I delete the DB?
                                                │
You reply: "No" ────────────────────▶ HeySummon ────▶ Agent gets "No"

Agent ──── "What should I do?" ──────▶ HeySummon
                                                │
                                      [HS-A1B2C3D4] What should I do?
                                                │
You reply: "Archive it instead" ─────▶ HeySummon ────▶ Agent gets the answer
```

## Setting up the watcher

Configure the expert watcher to receive notifications in your messaging app:

```env
# ~/clawd/skills/heysummon-expert/.env
HEYSUMMON_BASE_URL=http://localhost:3445
HEYSUMMON_API_KEY=hs_exp_abc123...
HEYSUMMON_NOTIFY_TARGET=<your_telegram_chat_id>
```

Start the watcher:

```bash
bash ~/clawd/skills/heysummon-expert/scripts/setup.sh
```

Check status:

```bash
pm2 logs heysummon-expert-watcher
```

---

## Tips

- **Ref codes are 8 chars** — always `HS-XXXXXXXX`
- **Reply quotes work** — Telegram's "Reply" feature auto-detects the ref code
- **Response is fast** — the agent picks up your answer on the next poll
- **Dashboard is always there** — fallback for when notifications don't work

<Callout type="info">
  The watcher uses `GET /api/v1/events/pending` on startup to catch up on any events missed while it was offline.
</Callout>
