# HeySummon

**The self-hosted, encrypted human-in-the-loop API for AI agents.**

Your AI agent asks a human for help — and waits for the answer.

---

## How it works

Your agent just says:

```
hey summon alice Should I delete these 2,400 records from production?
```

Alice gets a notification — on the dashboard, Telegram, Slack, or her phone.
She replies. The agent picks up the answer and continues.

That's it.

---

## Three steps

```
1. Agent asks       →  "hey summon alice Should I proceed?"
2. Human responds   →  Alice replies via dashboard, Telegram, or phone
3. Agent continues  →  picks up the answer and moves on
```

Every request gets a reference code like `HS-A1B2C3D4` so both sides can track it.

---

## When to use it

| Situation | Example |
|-----------|---------|
| **Budget approval** | Agent needs to spend $500 on a vendor invoice |
| **Judgment call** | Instructions are ambiguous, agent isn't sure what to do |
| **Design review** | Agent generated an image and wants a human to approve it |
| **Irreversible action** | Agent is about to delete data or send emails |
| **Compliance check** | Task involves personal data or legal content |

See [Summoning Guidelines](/best-practices) to learn how guidelines are configured during setup.

---

## What makes it different

- **Your data stays yours** — self-host on your infrastructure with one Docker command. Nothing phones home.
- **Encrypted end-to-end** — messages are encrypted from agent to human with X25519 + AES-256-GCM. The server stores only ciphertext it cannot read.
- **Works with any agent** — not tied to one framework. Claude Code, Codex, n8n, or any HTTP client.
- **Humans respond where they are** — dashboard, Telegram, Slack, WhatsApp, or phone. No context-switching.
- **Every request is signed and scanned** — the Guard proxy validates, rate-limits, and content-scans before anything reaches a human.
- **No infrastructure overhead** — simple HTTP polling, no WebSockets, no persistent connections. `npx heysummon` and you're running.
