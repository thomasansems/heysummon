# Self-Hosting Overview

HeySummon is designed to be self-hosted. Your data never leaves your infrastructure.

## Installation options

| Method | Best for | Setup time |
|--------|----------|------------|
| [NPX](/self-hosting/npx) | Personal use, evaluation | ~2 minutes |
| [Docker](/self-hosting/docker) | Production, teams | ~5 minutes |
| [From source](/getting-started/quickstart#option-3-local-development) | Development, contributing | ~10 minutes |

## Components

| Component | Description | Port |
|-----------|-------------|------|
| **App** | Next.js app — API, dashboard, auth, content safety | 3445 (Docker) |
| **PostgreSQL** | Database (Docker) or SQLite (NPX) | Internal only |

## Making it public

To expose HeySummon to the internet, choose one of:

| Option | Best for | What you need |
|--------|----------|---------------|
| **Caddy + HTTPS** *(recommended)* | Production on a VPS / EC2 | A domain you control |
| **Cloudflare Tunnel** | You already use Cloudflare DNS | A Cloudflare account |
| **Tailscale Funnel** | Free public URL, no domain | A Tailscale account |

The Docker installer ([install.sh](https://raw.githubusercontent.com/thomasansems/heysummon/main/install.sh)) walks you through the setup interactively, including DNS instructions for the Caddy option.

See [Docker → Making it public](/self-hosting/docker#making-it-public) for the full guide.
