# Polling for Updates

Poll the pending events endpoint to receive updates about your help requests.

---

## Poll for events

```bash
curl http://localhost:3425/api/v1/events/pending \
  -H "x-api-key: hs_live_abc123..."
```

Returns pending events as JSON:

```json
{
  "events": [
    {
      "type": "new_request",
      "requestId": "cmxxx...",
      "refCode": "HS-A1B2C3D4",
      "status": "pending",
      "createdAt": "2026-02-28T09:00:00Z"
    },
    {
      "type": "status_change",
      "requestId": "cmxxx...",
      "refCode": "HS-A1B2C3D4",
      "status": "responded",
      "respondedAt": "2026-02-28T09:05:00Z"
    }
  ]
}
```

---

## Event types

| Event | When | Payload |
|-------|------|---------|
| `new_request` | A help request is submitted | `requestId`, `refCode`, `question` |
| `status_change` | Status changes (pending -> responded, etc.) | `requestId`, `refCode`, `status` |
| `new_message` | A message is sent in a conversation | `requestId`, `refCode`, `messageId`, `from` |

## JavaScript example

```javascript
async function pollForUpdates(apiKey) {
  const response = await fetch('/api/v1/events/pending', {
    headers: { 'x-api-key': apiKey }
  });
  const { events } = await response.json();

  for (const event of events) {
    if (event.type === 'status_change' && event.status === 'responded') {
      console.log(`Request ${event.refCode} was answered!`);
      await fetchResponse(event.requestId);
    }
  }

  // Acknowledge processed events
  for (const event of events) {
    await fetch(`/api/v1/events/ack/${event.requestId}`, {
      method: 'POST',
      headers: { 'x-api-key': apiKey }
    });
  }
}

// Poll every 5 seconds
setInterval(() => pollForUpdates('hs_live_abc123...'), 5000);
```

---

## Acknowledging events

After processing events, acknowledge them so they are not returned again:

```bash
POST /api/v1/events/ack/:requestId
x-api-key: hs_live_abc123...
```

<Callout type="info">
  Pending events are stored for **24 hours** after the last activity on the request. Make sure to poll and acknowledge regularly.
</Callout>

---

## Polling tips

| Consideration | Recommendation |
|---|---|
| Poll interval | Every 3-10 seconds depending on urgency |
| Rate limits | 30 req/min per IP on polling endpoints |
| Missed events | Events are stored until acknowledged or expired |
| Best for | All agents, scripts, and one-off requests |
