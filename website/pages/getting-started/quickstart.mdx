# Quickstart

Get HeySummon running and send your first help request in under 5 minutes.

---

## Option 1: NPX (fastest)

```bash
npx @heysummon/app
```

The interactive installer handles everything: download, secrets, database, and starts the server on port 3435.

```bash
heysummon start       # start the server
heysummon start -d    # start in background (daemon)
heysummon stop        # stop the server
heysummon status      # check if running + show URL
heysummon update      # update to latest version
```

Jump to [Complete the onboarding](#complete-the-onboarding) once it's running.

---

## Option 2: Docker (recommended for self-hosting)

One command — downloads compose file, generates secrets, starts everything:

```bash
curl -fsSL https://raw.githubusercontent.com/thomasansems/heysummon/main/install.sh | bash
```

Installs to `~/.heysummon-docker/`. HeySummon is available at `http://localhost:3445`.

---

## Option 3: Local development

The setup wizard generates secrets, creates the database, and gets you ready to develop:

```bash
git clone https://github.com/thomasansems/heysummon.git
cd heysummon
bash install-local-dev.sh
pnpm dev
```

Open `http://localhost:3425` — sign up to create the admin account.

<Callout type="info">
  The wizard creates `.env.local` with auto-generated secrets and an SQLite database.
  To re-run, delete `.env.local` and run `bash install-local-dev.sh` again.
</Callout>

## Complete the onboarding

Once the server is running, open the dashboard:

| Install method | URL |
|---|---|
| NPX | `http://localhost:3435` |
| Docker | `http://localhost:3445` |
| Local dev | `http://localhost:3425` |

**Sign up** — the first user becomes the admin account. After that, HeySummon walks you through a guided onboarding:

### 1. Welcome

Quick intro to HeySummon and how the platform works.

### 2. Expert setup

Create your expert identity and choose how you want to receive notifications — Telegram, Slack, or OpenClaw.

### 3. Network

Configure public access so notification channels can reach your instance. This step is automatically skipped for polling-based channels like OpenClaw.

### 4. Test connection

Verify that notifications arrive on your chosen channel. HeySummon sends a test message and confirms delivery.

### 5. Client setup

Create your first API key and configure the client channel. This is the key your AI agents will use to send help requests.

### 6. End-to-end test

HeySummon runs a live end-to-end encryption test — sending a request, delivering it to you, and verifying the encrypted round-trip. You can watch the entire flow in real time.

### 7. Done

Everything is set up. You have an expert account, a notification channel, a client API key, and verified end-to-end encryption. Head to the dashboard to start receiving help requests from your AI agents.

<Callout type="info">
  You can skip the onboarding at any step and complete it later, or restart it from the beginning at any time.
</Callout>
