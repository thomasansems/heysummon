# Changelog

## Summoning Context -- 2026-03-30

### Added
- **Summoning context**: Providers can now set advisory guidelines (up to 500 characters) that tell AI agents when to summon them. Choose from presets (Strict, Budget-conscious, Safety-first) or write custom instructions.
- Summoning context is configurable in the onboarding wizard and the dashboard provider detail panel.
- Context flows through to the consumer's skill configuration as `HEYSUMMON_SUMMON_CONTEXT` environment variable.
- Both consumer and OpenClaw SKILL.md files include a new "When to summon" section that references the context.
- `GET /api/v1/providers` now includes `summonContext` in the response.
- Setup page shows summoning guidelines when configured.

## Docker Refactor — 2026-03-27

### Changed
- **Breaking:** Merged Guard content safety into the main app as built-in middleware. The separate `heysummon-guard` Docker image is no longer needed.
- Simplified Docker deployment: just app + PostgreSQL (no more Guard proxy, keygen service, or shared key volumes)
- Content safety (HTML sanitization, URL defanging, PII detection/blocking) now runs in-process on API routes
- Removed Ed25519 receipt signing system (`x-guard-receipt` headers no longer used)
- Single Docker image: `ghcr.io/thomasansems/heysummon`

### Removed
- `ghcr.io/thomasansems/heysummon-guard` Docker image (deprecated)
- `Dockerfile.allinone` (superseded by single Dockerfile)
- `docker-compose.prod.yml`, `docker-compose.cloudflare.yml`, `docker-compose.ngrok.yml`, `docker-compose.tailscale.yml` (superseded by profiles in `docker-compose.yml`)
- Environment variables: `GUARD_SIGNING_KEY`, `GUARD_PUBLIC_KEY`, `REQUIRE_GUARD`, `GUARD_PORT`
- `guard/` directory and `scripts/generate-guard-keys.ts`

### Migration
If upgrading from a previous version:
1. Pull the new `docker-compose.yml`
2. Remove `GUARD_*` and `REQUIRE_GUARD` from your `.env`
3. Run `docker compose up -d` — the guard-keygen and heysummon-guard services are no longer defined

## v0.1.0-beta — 2026-02-28

Initial public beta release.

### Features

**Platform**
- Help request submission and response flow
- Multi-turn conversations via `POST /api/v1/message`
- Reference codes (`HS-XXXXXXXX`) for every request
- Dashboard for human experts
- `GET /api/v1/events/pending` — poll for pending events
- HTTP polling for updates
- `POST /api/v1/events/ack/:id` — acknowledge delivery

**Security**
- Content safety scanning — XSS, PII (CC, SSN, BSN), malicious URLs
- OWASP ZAP automated scanning in CI (runs on every PR + nightly)
- Malicious content CI gate — build fails if content safety lets dangerous content through
- HMAC-SHA256 API key hashing (replaces plain SHA256)
- IP binding per key and per provider profile

**Auth & Keys**
- Client keys (`hs_live_*`) and provider keys (`hs_prov_*`)
- Key rotation with 24-hour grace period
- Device token binding
- IP Security inline on Clients and Users dashboard pages

**Infrastructure**
- Docker deployment with app and PostgreSQL
- NPX installer (`npx @heysummon/app`) for local/personal use
- GitHub Container Registry images at `ghcr.io/thomasansems/heysummon`
- Automated release workflow with Docker image publishing

**CLI**
- `npx @heysummon/app` installer
- `heysummon start/stop/status/update/logs`

**Documentation**
- Docs moved into main repo (`heysummon/website/`) — always in sync with code
- Automated Vercel deployment on every push to main
