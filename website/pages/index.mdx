# HeySummon

**HeySummon lets your AI agent ask a human for help — and wait for the answer.**

When your agent gets stuck, needs approval, or shouldn't act alone, it sends a request.
A human responds. The agent continues.

That's it.

---

## The basics

### Your agent sends a question

```bash
curl -X POST https://your-host/api/v1/help \
  -H "x-api-key: hs_live_..." \
  -H "Content-Type: application/json" \
  -d '{
    "question": "Should I delete these 2,400 records?",
    "agentName": "Cleanup Agent"
  }'
```

### It gets a reference code back

```json
{ "requestId": "cmxxx...", "refCode": "HS-A1B2C3D4", "status": "pending" }
```

### A human responds — via dashboard or Telegram

The human sees the question, replies, and the agent picks up the answer by polling.

```bash
curl https://your-host/api/v1/help/cmxxx... -H "x-api-key: hs_live_..."
# → { "status": "responded", "response": "No, do not delete." }
```

## Common use cases

| Situation | What the agent does |
|-----------|-------------------|
| Budget threshold reached | Asks for spending approval |
| Ambiguous instruction | Asks for a judgment call |
| Visual output before publishing | Asks for a design review |
| Irreversible action | Asks for explicit confirmation |
| No-spend constraint hit | Asks how to proceed |

---

## What makes it different

- **End-to-end encrypted** — the platform stores ciphertext it cannot read
- **No persistent connections** — agents poll via plain HTTP
- **Self-hostable** — one Docker command, your data stays on your infra
- **Multi-channel** — humans can respond via dashboard, Telegram, or any connected app
- **Guard proxy** — every request is content-scanned and Ed25519-signed
