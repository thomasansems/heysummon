# Changelog

## v0.1.0-alpha.2 — 2026-03-27

### Fixed
- API key generation now uses cryptographically secure random bytes (`crypto.randomBytes`) instead of `Math.random()`
- Reference code generation hardened with CSPRNG
- Removed insecure `"fallback-dev-secret"` from HMAC operations; server now refuses to start without `NEXTAUTH_SECRET`
- Key-exchange endpoint now requires session authentication and verifies the caller is the assigned provider

---

## Infrastructure — 2026-03-26

### Changed
- Docs deployment to Vercel now happens explicitly via GitHub Actions (Vercel CLI) instead of Vercel's automatic GitHub integration, triggered on push to `main` with changes in `/website` or `/docs`

---

## Documentation — 2026-03-26

### Changed
- Consolidated documentation: `/website/pages/` is the single source of truth, `/docs/` marked as legacy
- Rewrote homepage with paperclip-inspired structure — simple intro, then deep-dive links
- Rewrote best-practices page with 5 copy-paste system prompts using `hey summon NAME` pattern

## v0.1.0-beta — 2026-02-28

Initial public beta release.

### Features

**Platform**
- Help request submission and response flow
- Multi-turn conversations via `POST /api/v1/message`
- Reference codes (`HS-XXXXXXXX`) for every request
- Dashboard for human experts
- `GET /api/v1/events/pending` — poll for pending events
- HTTP polling for updates
- `POST /api/v1/events/ack/:id` — acknowledge delivery

**Security**
- Guard proxy — Ed25519-signed receipts on every request
- Content safety scanning — XSS, PII (CC, SSN, BSN), malicious URLs
- OWASP ZAP automated scanning in CI (runs on every PR + nightly)
- Malicious content CI gate — build fails if Guard lets dangerous content through
- HMAC-SHA256 API key hashing (replaces plain SHA256)
- IP binding per key and per provider profile

**Auth & Keys**
- Client keys (`hs_live_*`) and provider keys (`hs_prov_*`)
- Key rotation with 24-hour grace period
- Device token binding
- IP Security inline on Clients and Users dashboard pages

**Infrastructure**
- Docker deployment with Guard, Platform, PostgreSQL
- NPX installer (`npx heysummon`) for local/personal use
- GitHub Container Registry images at `ghcr.io/thomasansems/heysummon`
- Automated release workflow with Docker image publishing

**CLI**
- `npx heysummon` installer
- `heysummon start/stop/status/update/logs`

**Documentation**
- Docs moved into main repo (`heysummon/website/`) — always in sync with code
- Automated Vercel deployment on every push to main
