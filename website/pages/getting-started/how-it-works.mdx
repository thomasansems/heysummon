# How It Works

HeySummon is a relay platform that connects AI agents to human experts in real time.

---

## Request lifecycle

```
pending  →  reviewing  →  responded
    └──────────────────────→  expired  (after TTL, default 72h)
                         →  closed    (manually via API)
```

| Status | Meaning |
|--------|---------|
| `pending` | Submitted, waiting for a human to open it |
| `reviewing` | Human has opened the request in the dashboard |
| `responded` | Human has replied |
| `expired` | TTL elapsed (default 72 hours) |
| `closed` | Manually closed via `/api/v1/close/:id` |

---

## Full flow

```
AI Agent
  │
  │  POST /api/v1/help
  │  { apiKey, question (encrypted), signPublicKey, encryptPublicKey }
  ▼
App
  │  Validates x-api-key
  │  Content safety: XSS, PII, malicious URLs
  │  Generates ref code: HS-XXXXXXXX
  │  Stores encrypted request
  │  Creates event on provider topic
  ▼
Provider dashboard / watcher (via polling)
                       │
                       │  Notification: "🦞 [HS-XXXXXXXX] New request"
                       │  Human reads, responds
                       ▼
                   Platform
                       │  Status → responded
                       │  Creates event on consumer topic
                       ▼
                   AI Agent (via polling)
                       │  GET /api/v1/help/:id → { status: "responded" }
```

---

## Reference codes

Every request gets an `HS-XXXXXXXX` code (8 alphanumeric characters).

These codes are:
- Shown in the dashboard and notifications
- Used to reply from a messaging app: `[HS-A1B2C3D4] Yes, proceed`
- Included in audit logs for traceability

---

## Real-time vs polling

| Method | How | Latency |
|--------|-----|---------|
| **Event polling** | `GET /api/v1/events/pending` — poll for new events | Seconds |
| **Status polling** | `GET /api/v1/help/:id` — check every N seconds | N seconds |

Use event polling for production agents. Status polling is simpler for scripts and one-off requests.

---

## Multi-turn conversations

Requests support back-and-forth messages. After the initial response, the agent can send follow-up questions using `POST /api/v1/message/:requestId`. Each message gets a Mercure notification.

All messages in a conversation share the same `HS-XXXXXXXX` ref code.

---

## Content safety

All API requests are automatically scanned by built-in content safety middleware:

- Scans for malicious content: XSS, HTML injection, PII (credit cards, SSNs, BSNs)
- Defangs suspicious URLs
- Blocks requests containing credit cards or SSN/BSN patterns (422)

See [Security → Content Safety](/security/content-safety-guard) for details.
