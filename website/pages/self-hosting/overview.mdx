# Self-Hosting Overview

HeySummon is designed to be self-hosted. Your data never leaves your infrastructure.

## Installation options

| Method | Best for | Setup time |
|--------|----------|------------|
| [NPX](/self-hosting/npx) | Personal use, evaluation | ~2 minutes |
| [Docker](/self-hosting/docker) | Production, teams | ~5 minutes |
| [From source](/getting-started/quickstart#option-3-local-development) | Development, contributing | ~10 minutes |

## Components

| Component | Description | Port |
|-----------|-------------|------|
| **App** | Next.js app — API, dashboard, auth, content safety | 3445 (Docker) |
| **PostgreSQL** | Database (Docker) or SQLite (NPX) | Internal only |

## Making it public

To expose HeySummon to the internet, use a tunnel or reverse proxy:

- **Cloudflare Tunnel** — recommended, free
- **Tailscale Funnel** — good for team setups
- **Nginx / Caddy** — for VPS deployments

See [Docker](/self-hosting/docker#making-it-public) for tunnel configuration.
