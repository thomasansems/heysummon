# Summoning Guidelines

HeySummon uses a **Summoning Wizard** to generate personalized guidelines for your AI
agents. Instead of manually writing system prompts, you configure how and when the agent
should ask a human for help -- and HeySummon generates the instructions automatically.

<Callout type="info">
  Guidelines are configured during **onboarding** (step 4) or when **adding a new client**
  from the dashboard. You can also reuse previously created guidelines.
</Callout>

---

## How it works

<Steps>

### Set autonomy level

Choose how independently the AI agent should operate:

| Level | Behavior |
|-------|----------|
| **Autonomous** | Operates independently. Only summons the expert when completely stuck. |
| **Balanced** | Works autonomously on routine tasks. Summons for important decisions or ambiguous requirements. |
| **Conservative** | Summons before any risky, unclear, or potentially impactful action. Prefers caution over speed. |
| **Strict** | Summons before most non-trivial actions. Only proceeds autonomously on simple, well-defined, low-risk tasks. |

### Configure safety gates

Safety gates are hard rules -- the agent will **always** summon a human before these
actions, regardless of autonomy level:

- **Destructive actions** -- deleting data, dropping tables, removing files
- **Financial decisions** -- purchases, billing changes, cost-affecting actions
- **Permission changes** -- modifying access, roles, or security settings
- **External communications** -- sending emails, messages, or notifications
- **Deployments** -- deploying to production or staging environments

By default, *destructive actions* and *financial decisions* are enabled.

### Define expert strengths

Tell the agent what the human expert is especially good at, so it knows when human
input adds the most value:

- **Code review** -- architecture decisions and code quality
- **Emotional intelligence** -- sensitive situations requiring empathy
- **Domain expertise** -- industry-specific knowledge and context
- **Creative decisions** -- design, branding, and creative direction
- **Research** -- deep research and investigation tasks
- **Legal & compliance** -- legal review, regulatory matters

### Choose timeout fallback

Decide what the agent should do if no expert responds within the timeout period:

| Fallback | Behavior |
|----------|----------|
| **Proceed cautiously** | Pick the safest available option. Document all decisions made without expert input and flag them for review. |
| **Skip & continue** | Skip the blocked task, continue with other available work, and flag the skipped task for expert review later. |
| **Stop entirely** | Halt all execution immediately until an expert responds. |

</Steps>

## What gets generated

After completing the wizard, HeySummon generates a set of markdown-formatted instructions
that get embedded into the AI agent's configuration. Here is an example:

```markdown
## Summoning Guidelines

### Autonomy Level: Balanced
Work autonomously on routine tasks. Summon the expert for important
decisions, ambiguous requirements, or when multiple valid approaches exist.

### Safety Gates (Always Summon)
- Always summon before destructive actions (deleting data, dropping tables, removing files)
- Always summon before financial decisions (purchases, billing changes, cost-affecting actions)

### Expert Strengths (Prefer Human Input)
- Prefer human input for code review, architecture decisions, and code quality

### Timeout Fallback
If no expert responds within the timeout, proceed with the safest available
option. Document all decisions made without expert input and flag them for review.
```

The agent receives these guidelines automatically when it is set up via the setup link.
No manual prompt editing required.

---

## Where guidelines are configured

You can create or update guidelines in two places:

1. **Onboarding** -- when you first connect a client (step 4 of the setup flow)
2. **Dashboard** -- when adding a new client under *Clients*

HeySummon remembers your most recent guidelines per expert, so the next time you set up
a client, your previous configuration is pre-filled.

---

## How guidelines reach the agent

When you complete the wizard and create a client connection:

1. HeySummon generates a **setup link** (valid for 24 hours)
2. The setup link contains an install command with guidelines embedded
3. Running the install command configures the AI agent's environment
4. The agent's HeySummon skill reads the guidelines and follows them during operation

The agent does not need to know about API endpoints, keys, or encryption -- the
HeySummon skill handles all of that.

## Tips

- **Start with Balanced.** It is the default for a reason -- most teams want agents that
  can work independently but check in on important calls.
- **Enable safety gates for your risk profile.** If your agent handles payments, enable
  *financial decisions*. If it manages infrastructure, enable *deployments*.
- **Add expert strengths sparingly.** Only mark strengths where human input genuinely
  outperforms the AI -- this keeps summon frequency manageable.
- **Use "Stop entirely" for high-stakes environments.** If a wrong autonomous decision
  could cause real damage, stopping is safer than proceeding.
- **Reuse guidelines across clients.** HeySummon saves your recent configurations, so
  you can quickly apply the same guidelines to new agents.
