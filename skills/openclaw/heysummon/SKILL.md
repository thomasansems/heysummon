# HeySummon — Human in the Loop

When you need human input, approval, or are stuck on something that requires human judgment — use HeySummon to ask a human expert and **wait for their response** before continuing.

## Setup

Run the setup script, which detects OpenClaw and configures keypairs, hooks, and the watcher automatically:

```bash
bash {baseDir}/scripts/setup.sh
```

This will:
1. Prompt for your HeySummon base URL and API key
2. Validate the API key
3. Register the provider
4. Generate keypairs in `.keys/`
5. Generate and register a hooks token in `~/.openclaw/openclaw.json`
6. Install the `heysummon-responder` hook to `~/.openclaw/hooks/`
7. Start the background watcher via PM2 (`heysummon-watcher`)

After setup, restart the OpenClaw gateway to activate hooks:

```bash
openclaw gateway restart
```

## How to ask

```bash
# Ask a specific provider (recommended)
bash {baseDir}/scripts/ask.sh "Your question here" "" "ProviderName"

# Ask without specifying a provider (uses default)
bash {baseDir}/scripts/ask.sh "Your question here"

# Ask with context messages
bash {baseDir}/scripts/ask.sh "Is this the right approach?" '[{"role":"user","content":"context"}]' "ProviderName"
```

The script **blocks and returns the human's answer** on stdout. If the timeout is reached, the background watcher will still capture the response.

### Async mode (non-blocking)

```bash
# Submit and return immediately
bash {baseDir}/scripts/ask.sh --async "Your question here" "" "ProviderName"

# Check for responses later
bash {baseDir}/scripts/ask.sh --check
```

## Other commands

```bash
# Add a provider
bash {baseDir}/scripts/add-provider.sh "hs_cli_key" "FriendlyName"

# List registered providers
bash {baseDir}/scripts/list-providers.sh

# Check request status
bash {baseDir}/scripts/check-status.sh <refCode|requestId>

# Watcher management
bash {baseDir}/scripts/openclaw-setup.sh          # Full OpenClaw setup (keypairs + hooks + watcher)
bash {baseDir}/scripts/teardown.sh                 # Stop the watcher
```

## Event flow (OpenClaw)

```
Provider responds (Telegram)
    ↓
Platform watcher polls /api/v1/events/pending
    ↓
notify.sh → POST /hooks/agent (wakes agent in existing session)
    ↓
heysummon-responder hook detects response notification
    ↓
Agent resumes with full conversation context
```

## Handling responses

- **Normal response**: The provider's answer is returned on stdout. Use it to continue your work.
- **Provider unavailable**: The request is queued and will be delivered when the provider comes online. Keep waiting for the response.
- **`TIMEOUT`**: No response within 15 minutes. The watcher will capture it later — check with `ask.sh --check`.

## Rules

- **Always wait** for the response before continuing (in blocking mode)
- **Be specific** — include relevant context in your question
- **Don't spam** — one request at a time; don't ask trivial questions

## Configuration (.env)

```env
HEYSUMMON_BASE_URL=http://localhost:3425
HEYSUMMON_API_KEY=hs_cli_your_key_here
HEYSUMMON_TIMEOUT=900
HEYSUMMON_POLL_INTERVAL=3
HEYSUMMON_NOTIFY_MODE=message
HEYSUMMON_NOTIFY_TARGET=<your-chat-id>
HEYSUMMON_AGENT_ID=tertiary
HEYSUMMON_HOOKS_TOKEN=<auto-generated>
HEYSUMMON_SESSION_KEY=<auto-detected>
```
